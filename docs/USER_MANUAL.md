# Discipline Management System: Staff User Manual

## 1) What This System Is For
This system helps staff:
- record student discipline faults
- manage permission requests
- track conduct scores
- view reports and dashboard insights

## 2) How To Access
1. Open the system link in your browser.
2. Enter your username and password.
3. Click the eye icon in password field if you want to view what you typed.
4. Click `Login`.

## 3) User Roles
- `Teacher`: records faults and submits related requests.
- `Admin`: manages permissions, student records, and operations.
- `Director`: monitors dashboards, reports, settings, and staff management.

## 4) Common Daily Tasks

### A) Teacher: Record a Fault
1. Open Teacher Dashboard.
2. Search/select the student.
3. Choose fault type and points.
4. Submit.

Result:
- Student conduct score is updated based on points deducted.

### B) Admin: Review Permissions
1. Open Permissions page.
2. Use filters (status/date/student).
3. Open a request.
4. Approve or deny with reason when needed.

### C) Director: View School Discipline Status
1. Open Director Dashboard.
2. Review totals (students, teachers, faults, pending requests).
3. Open analytics/reports for class trends and behavior patterns.

### D) Register a New Teacher (Director)
1. Go to `Register Teacher`.
2. Enter name/credentials.
3. Save.

## 5) Navigation Tips
- Use sidebar menu for modules:
  - Dashboard
  - Students
  - Permissions
  - Reports
  - Settings
- Use search boxes to quickly find students.
- Use filters before exporting/reporting.

## 6) Recommended Operating Routine
- Teachers: submit faults on the same day.
- Admin: review permissions daily.
- Director: check dashboard weekly and reports monthly.

## 7) If Something Looks Wrong

### Login fails
- Recheck username/password (case sensitive).
- Use password eye toggle to confirm typing.

### Page does not load data
- Refresh page.
- Check internet connection.
- Log out and log in again.

### Changes not visible immediately
- Wait a few seconds and refresh.
- If hosted on free tier, first load can be slow after inactivity.

## 8) Data Safety Rules
- Do not share passwords.
- Do not leave account logged in on shared computers.
- Always log out after use.
- Report suspicious activity to system owner immediately.
